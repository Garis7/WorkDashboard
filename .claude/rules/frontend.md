# フロントエンド (htmx + Jinja2 + CSS)

htmx によるサーバーサイドレンダリングを基本とする。

## 原則
- インラインスタイルは使わず CSS クラスに統一する
- CSS フレームワークのデフォルトスタイルを理解してから実装する
- テンプレートで `.label` / `.value` を呼ぶ箇所には null guard を入れる
- htmx リクエスト判定で部分テンプレート / フルページを切り替える
- CSS クラス名は用途ベースで付ける
- 色・サイズなどのマジックナンバーは CSS 変数か共通クラスに寄せる
- システム状態の可視性、一貫性、再認性、最小限のデザインを意識する

## 落とし穴
- PicoCSS の `<button>` は白文字前提のため、カスタム時は `color` を明示する
- htmx 2.x は `hx-boost` の挙動差分に注意する
- enum の `.value` を CSS クラス名に使う場合は `_` を `-` に変換する
